# WorkTime Sync

**WorkTime Sync** — веб-система для актуализации рабочего времени сотрудников. Проект помогает видеть реальные рабочие графики, конфликты с календарём, перегрузки, расхождения с HR-данными, статусы подтверждения графиков и задачи, которые нужно выполнить для устранения проблем.

Проект разработан для кейса №3 хакатона **«Весенний код»**: **«Система по актуализации рабочего времени сотрудников»**.

---

## 1. Суть проекта

В компаниях рабочее время сотрудников часто хранится в разных местах: HR-системе, календарях, таск-трекерах, табелях и личных договорённостях. Эти данные быстро устаревают: сотрудники меняют формат работы, переходят на удалёнку или гибрид, работают из другого часового пояса, уходят в отпуск или получают встречи вне рабочего времени.

**WorkTime Sync** решает эту проблему как аналитический и workflow-контур:

1. собирает или имитирует данные о сотрудниках, графиках, встречах, HR-профилях и отсутствиях;
2. сравнивает заявленный график с фактической активностью;
3. рассчитывает актуальность графика, загрузку, конфликты и риск;
4. показывает проблемные зоны руководителям и HR;
5. позволяет создавать задачи сотрудникам;
6. позволяет сотрудникам подтверждать график или отвечать на задачи;
7. позволяет руководителю/HR применить решение;
8. обновляет связанные данные, например время встречи после согласованного переноса.

Главная логика:

```text
аналитика → задача → ответ сотрудника → применение решения → обновление данных
```

## 3. Ролевая модель

| Роль | Scope | Что видит | Что может делать |
|---|---|---|---|
| `executive` | `all` | всю компанию, все отделы, всех сотрудников | смотреть общую аналитику, контролировать отделы, создавать задачи любому сотруднику, применять решения |
| `hr` | `all` | всех сотрудников, HR-расхождения, подтверждения | создавать HR-задачи, проверять графики и HR-данные, применять решения |
| `department_manager` | `department` | только сотрудников своего отдела | создавать задачи сотрудникам своего отдела, контролировать графики, встречи и нагрузку, применять решения |
| `employee` | `self` | только свой профиль и свои задачи | подтверждать график, подтверждать/отклонять задачи, предлагать перенос, оставлять комментарии |

## 6. Задачи в системе

| Тип | Что значит |
|---|---|
| `confirm_schedule` | подтвердить рабочий график |
| `review_hr_profile` | проверить HR-профиль |
| `review_load` | проверить нагрузку |
| `update_timezone` | проверить или обновить часовой пояс |
| `meeting_confirmation` | подтвердить участие во встрече |
| `reschedule_meeting` | согласовать перенос встречи |
| `meeting_outside_work_approval` | согласовать встречу вне рабочего времени |

Статусы: `pending`, `confirmed`, `rejected`, `done`, `expired`, `reschedule_requested`.

Важно:

```text
events.csv  → сама встреча
tasks.json  → подтвердить / перенести / согласовать эту встречу
```

## 7. Формулы и метрики

Актуальность графика:

```text
A = max(0, 1 - days_since_update / 90)
```

Интегральный риск:

```text
R = wA * (1 - A) + wC * C + wL * L + wT * timezone_mismatch + wH * hr_mismatch
```

Система считает актуальность графика, дни с последнего обновления, долю встреч вне графика, конфликты, загрузку, командную доступность, интегральный риск и лучшие окна встреч.

## 10. Локальный запуск

Backend:

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Frontend:

```bash
cd frontend
npm install
npm run dev
```

Для локального подключения к backend:

```env
VITE_API_BASE_URL=http://127.0.0.1:8000
VITE_USE_MOCK_DATA=false
```

Для deploy на одном домене можно оставить `VITE_API_BASE_URL=` и использовать `/api/...`.

## 11. Production/deploy заметки

Nginx:

```nginx
location / {
    try_files $uri $uri/ /index.html;
}

location /api/ {
    proxy_pass http://127.0.0.1:8000/api/;
}
```

Backend должен иметь права записи в активную папку данных, иначе создание задач, подтверждения графиков и применение переносов не сохранятся.

## 12. Демонстрационный сценарий

1. Войти как `zarix / i9VUibm6`.
2. Открыть сотрудников отдела Core Platform.
3. Найти встречу вне рабочего времени.
4. Создать задачу на перенос/согласование.
5. Войти как `gleb_employee / emp5`.
6. Открыть личный кабинет.
7. Ответить на задачу.
8. Вернуться как руководитель.
9. Нажать “Применить решение”.
10. Проверить, что задача закрылась, а событие обновилось.
