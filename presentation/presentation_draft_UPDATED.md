# Черновик презентации WorkTime Sync

## Слайд 1. Титульный слайд

**WorkTime Sync**  
Система по актуализации рабочего времени сотрудников

Кейс №3 хакатона «Весенний код»

Ключевой тезис:

> WorkTime Sync помогает видеть актуальность графиков, конфликты с календарём, перегрузку, расхождения с HR-данными и лучшие окна для встреч.

---

## Слайд 2. Проблема

Рабочие графики сотрудников часто становятся неактуальными:

- сотрудники переходят на удалёнку или гибрид;
- меняются часовые пояса;
- появляются командировки, отпуска, больничные и личные часы;
- HR-данные, календарь и реальная занятость расходятся;
- встречи назначаются вне рабочего времени;
- руководители не видят реальную доступность команды.

Итог: компания планирует работу по устаревшим данным.

---

## Слайд 3. Идея решения

WorkTime Sync — веб-система, которая анализирует рабочее время сотрудников и помогает поддерживать его в актуальном состоянии.

Система сравнивает:

- заявленный график;
- календарные события;
- HR-профиль;
- временные исключения;
- загрузку;
- дату последнего обновления.

После анализа система считает метрики, риск и рекомендации.

---

## Слайд 4. Роли пользователей

**Сотрудник**  
Подтверждает или обновляет рабочий график.

**HR**  
Проверяет корректность HR-профилей, форматов работы и часовых поясов.

**Руководитель**  
Видит состояние команды, риски и перегрузку.

**Проектный менеджер**  
Выбирает лучшие окна для встреч и планирует задачи.

**Администратор**  
Настраивает источники данных и загрузку файлов.

**Аналитик**  
Изучает статистику конфликтов, доступности и актуальности графиков.

## Слайд 5. Что показывает MVP

В MVP есть frontend и backend.

Frontend показывает:

- дашборд;
- сотрудников;
- карточку сотрудника;
- конфликты;
- командную доступность;
- рекомендации.

Backend отдаёт:

- общий overview для frontend;
- список сотрудников;
- карточку сотрудника;
- summary команды;
- конфликты;
- расхождения данных;

- рекомендации.

## Слайд 6. Данные

Сейчас используются синтетические JSON/CSV-данные.

Файлы backend:

```text
backend/data/employees.json
backend/data/events.json
backend/data/hr_profiles.json
backend/data/absences.json
```

В текущем `employees.json` — 6 сотрудников.

Важно: количество сотрудников и команд не захардкожено. Backend читает файл данных и динамически считает:

- количество сотрудников;
- команды/отделы;
- средний риск;
- конфликты;
- доступность.

## Слайд 9. Формулы

### Дни с последнего обновления

```text
days_since_update = today - last_update_date
```

### Актуальность графика

```text
schedule_actuality = max(0, 1 - days_since_update / 90)
```

### Доля встреч вне рабочего времени

```text
outside_work_ratio = outside_work_events / total_events
```

### Загрузка

```text
load = busy_hours / work_hours
```

### Риск

```text
risk = 0.30 * (1 - schedule_actuality)
     + 0.25 * outside_work_ratio
     + 0.25 * load
     + 0.10 * timezone_mismatch
     + 0.10 * hr_mismatch
```

## Слайд 10. Командная доступность

Backend строит карту доступности команды.

Учитываются:

- рабочие дни;
- рабочие часы;
- календарные события;
- отпуска;
- личные часы;
- командировки;
- перегруз.

На основе этой карты система выбирает лучшие окна для встречи.

---

## Слайд 11. Рекомендации

Система формирует объяснимые рекомендации.

Примеры:

- попросить сотрудника обновить график;
- перенести встречу вне рабочего времени;
- проверить часовой пояс;
- сверить HR-профиль;
- не назначать новые встречи перегруженному сотруднику;
- использовать рекомендованный командный слот.

WorkTime Sync помогает планировать работу по реальной доступности сотрудников, а не по устаревшим записям.

- меньше встреч вне рабочего времени;

- понятнее загрузка команды;
- быстрее актуализация HR-данных;
- прозрачнее планирование встреч и задач.
