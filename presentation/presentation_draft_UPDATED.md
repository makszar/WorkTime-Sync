# Черновик презентации WorkTime Sync

## Слайд 1. Титульный слайд

**WorkTime Sync**  
Система актуализации рабочего времени сотрудников

Кейс №3 хакатона «Весенний код»

> Рабочий график должен стать живым источником планирования, а не устаревшей записью в HR-системе.

---

## Слайд 2. Проблема

Рабочие графики сотрудников быстро устаревают.

| Причина | Последствие |
|---|---|
| сотрудник сменил формат работы | HR-данные и фактическая работа расходятся |
| сотрудник работает из другого часового пояса | встречи попадают в неудобное время |
| появились отпуска, больничные, day off | команда не учитывает отсутствие |
| календарь перегружен | растут переработки |
| график давно не подтверждался | руководитель не уверен в актуальности данных |

---

## Слайд 3. Идея решения

WorkTime Sync анализирует данные о рабочем времени и помогает актуализировать их через роли и действия.

```text
данные → метрики → риск → задача → подтверждение → актуализация
```

Система не просто показывает проблему, а помогает её закрыть: руководитель или HR создаёт задачу, сотрудник подтверждает график или оставляет комментарий.

---

## Слайд 4. Роли пользователей

| Роль | Что видит | Что делает |
|---|---|---|
| Полный руководитель | всю компанию | сравнивает отделы и общие риски |
| Руководитель отдела | свой отдел | создаёт задачи и контролирует сотрудников |
| HR | всех сотрудников с HR-фокусом | проверяет графики, HR-расхождения, подтверждения |
| Сотрудник | только себя | подтверждает график и отвечает на задачи |

   ↓

   ↓

   ↓

```text

   ↓
разный интерфейс под роль
   ↓
tasks API
   ↓
личный кабинет сотрудника
   ↓
подтверждение графика
```

## Слайд 9. Формулы

Актуальность графика остаётся одинаковой для всех:

```text
A = max(0, 1 - days_since_update / 90)
```

Риск:

```text
R = wA*(1-A) + wC*C + wL*L + wT*T + wH*H
```

Смысл: формула одна и понятная, но веса можно адаптировать под специфику отдела.

## Слайд 11. Личный кабинет сотрудника

В личном кабинете сотрудник видит:

- свой график;
- свой часовой пояс;
- формат работы;
- актуальность графика;
- риск;
- свои конфликты;
- задачи от руководителя/HR.

Действия сотрудника:

- подтвердить график;
- принять задачу;
- отклонить задачу с комментарием;
- запросить изменение.

## Слайд 17. Финальный вывод

WorkTime Sync превращает актуализацию рабочего времени из ручной сверки в управляемый процесс.

Система помогает находить устаревшие графики, видеть конфликты и переработки, учитывать отсутствия, планировать встречи, создавать задачи, получать подтверждение от сотрудников и снижать риски неактуальных данных.
