# Черновик презентации WorkTime Sync

## Слайд 1. Титульный слайд

**WorkTime Sync**  
Система актуализации рабочего времени сотрудников

Кейс №3 хакатона «Весенний код»

---

## Слайд 2. Проблема

Рабочие графики быстро устаревают.

| Проблема | Последствие |
|---|---|
| сотрудник сменил формат работы | HR-данные расходятся с реальностью |
| сменился часовой пояс | встречи попадают в неудобное время |
| есть отпуска и больничные | команда не учитывает отсутствие |
| календарь перегружен | появляются переработки |
| график давно не подтверждался | руководитель не уверен в данных |

---

## Слайд 3. Идея решения

WorkTime Sync превращает актуализацию рабочего времени в управляемый процесс.

```text
данные → аналитика → риск → задача → подтверждение → актуализация
```

Система не только показывает проблему, но и помогает её закрыть через задачи и личные кабинеты.

## Слайд 8. Формула риска

Актуальность:

```text
A = max(0, 1 - days_since_update / 90)
```

Риск:

```text
R = wA*(1-A) + wC*C + wL*L + wT*T + wH*H
```

Веса зависят от отдела, но структура формулы остаётся простой.

---

## Слайд 10. Задачи по встречам

Встреча хранится отдельно, задача — отдельно.

```text
events.csv  → сама встреча
tasks.json  → действие по встрече
```

Так можно назначить сотруднику задачу: подтвердить участие, согласовать перенос или подтвердить встречу вне рабочего времени.

---

## Слайд 11. Личный кабинет сотрудника

Сотрудник должен видеть:

- свой график;
- свою актуальность;
- свою загрузку;
- свои встречи;
- свои задачи;
- статус подтверждения графика.

Действия: подтвердить график, подтвердить задачу, отклонить задачу с комментарием, запросить изменение.

## Слайд 16. Вывод

WorkTime Sync — это система, которая помогает не просто найти проблему с рабочим временем, а довести её до решения.

```text
риск найден → задача создана → сотрудник подтвердил → данные стали актуальнее
```
