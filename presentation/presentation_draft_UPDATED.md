# Черновик презентации WorkTime Sync

## Слайд 1. Титульный слайд

**WorkTime Sync**  
Система по актуализации рабочего времени сотрудников

Кейс №3 хакатона «Весенний код»

Короткий тезис:

> WorkTime Sync помогает компаниям видеть актуальность рабочих графиков, конфликты с календарём, перегрузку сотрудников и рекомендации по обновлению данных.

---

## Слайд 2. Проблема

В компаниях рабочие графики сотрудников часто становятся неактуальными.

Причины:

- сотрудники переходят на удалённый или гибридный формат;
- меняются часовые пояса;
- появляются отпуска, больничные, командировки и личные часы;
- данные хранятся в разных системах;
- HR-профиль, календарь и реальная занятость могут расходиться.

Из-за этого:

- встречи назначаются вне рабочего времени;
- руководители не видят реальную доступность команды;
- проектные менеджеры планируют задачи без учёта перегрузки;
- HR тратит время на ручную сверку данных.

---

## Слайд 3. Идея решения

WorkTime Sync — это веб-система, которая анализирует рабочее время сотрудников и помогает поддерживать его в актуальном состоянии.

Система не просто хранит график, а сравнивает:

- заявленный график;
- календарные события;
- HR-профиль;
- занятые часы;
- часовой пояс;
- дату последнего обновления.

На основе этих данных система считает метрики и формирует рекомендации.

## Слайд 4. Роли пользователей

### Сотрудник

Обновляет рабочий график, подтверждает актуальность, видит личные рекомендации.

### HR-специалист

Следит за корректностью HR-данных, графиков, форматов работы и часовых поясов.

### Руководитель

Видит состояние команды, перегрузку, риски и сотрудников, которым нужно обновить график.

### Проектный менеджер

Планирует встречи и задачи с учётом реальной доступности.

### Администратор

Настраивает источники данных, пользователей и права.

### Аналитик

Изучает статистику, конфликты и качество планирования.

## Слайд 9. Формулы

### Дни с последнего обновления

```text
days_since_update = today - last_update_date
```

### Актуальность графика

```text
schedule_actuality = max(0, 1 - days_since_update / 90)
```

### Доля событий вне рабочего времени

```text
outside_work_ratio = outside_work_events / total_events
```

### Загрузка

```text
load = busy_hours / work_hours
```

### Риск

```text
risk = 0.30 * (1 - schedule_actuality)
     + 0.25 * outside_work_ratio
     + 0.25 * load
     + 0.10 * timezone_mismatch
     + 0.10 * hr_mismatch
```

## Слайд 10. Система рекомендаций

Система формирует рекомендации на основе рассчитанных метрик.

Примеры:

- обновить рабочий график, если данные давно не подтверждались;
- проверить встречи вне рабочего времени;
- проверить загрузку, если календарь близок к перегрузке;
- сверить часовой пояс с HR-профилем;
- согласовать рабочий формат и часы с HR-данными.

Главное: каждая рекомендация объяснима, потому что связана с конкретной метрикой.

## Слайд 15. Финальный вывод

WorkTime Sync помогает компании планировать работу на основе реальной доступности сотрудников, а не устаревших записей в разных системах.
