# Что отсутствует или требует уточнения по текущему состоянию WorkTime Sync

## Уже реализовано

### Frontend

- React + Vite приложение.
- Дашборд.
- Список сотрудников.
- Карточка сотрудника.
- Конфликты.
- Командная доступность.
- Рекомендации.
- Моковые данные для автономного демо.
- Расчёты во frontend: актуальность, риск, загрузка, доступность, лучшие слоты, рекомендации.

### Backend

- FastAPI backend MVP.
- Чтение данных из JSON.
- Подготовлена функция чтения JSON/CSV.
- Endpoint `GET /`.
- Endpoint `GET /employees`.
- Endpoint `GET /employees/{employee_id}`.
- Endpoint `GET /analytics/summary`.
- Расчёт метрик:
  - `days_since_update`;
  - `schedule_actuality`;
  - `outside_work_ratio`;
  - `load`;
  - `risk`;
  - `risk_status`;
  - `conflict_count`.
- Генерация рекомендаций в карточке сотрудника.
- Синтетические данные:
  - `employees.json`;
  - `events.json`;
  - `hr_profiles.json`.

## Что отсутствует

### 1. Полная связка frontend и backend

Frontend API-слой ожидает:

```http
GET /api/worktime/overview
```

Backend сейчас отдаёт:

```http
GET /employees
GET /employees/{employee_id}
GET /analytics/summary
```

Нужно либо добавить `/api/worktime/overview`, либо изменить frontend API-слой под текущие endpoint'ы backend.

### 2. Backend endpoint для конфликтов

Пока нет отдельного endpoint:

```http
GET /analytics/conflicts
```

Конфликты считаются внутри метрик сотрудника, но нет отдельной страницы/API-выдачи всех конфликтов со списком событий.

### 3. Backend endpoint для командной доступности

Пока нет endpoint:

```http
GET /analytics/availability
```

Командная доступность реализована во frontend на моковых данных.

### 4. Backend endpoint для лучших окон встречи

Пока нет endpoint:

```http
GET /meeting-slots
```

Подбор лучших окон есть во frontend на mockData, но backend пока его не отдаёт.

### 5. Общий endpoint для рекомендаций

Пока нет endpoint:

```http
GET /recommendations
```

Рекомендации формируются в карточке конкретного сотрудника, но нет общего списка всех рекомендаций для HR/руководителя.

### 6. Загрузка файлов через API

Пока нет endpoint:

```http
POST /upload
```

При этом функция чтения CSV/JSON уже подготовлена на уровне `load_table()`.

### 7. База данных

Сейчас данные лежат в JSON-файлах. Для полноценной системы нужна база данных:

- PostgreSQL;
- SQLite;
- или другая БД для MVP+.

### 8. Авторизация и роли

В кейсе предусмотрены роли:

- сотрудник;
- HR;
- руководитель;
- PM;
- администратор;
- аналитик.

В текущем коде авторизация и разграничение прав пока не реализованы.

### 9. Реальные интеграции

Пока нет реальных интеграций с:

- Google Calendar;
- HRM;
- таск-трекером;
- табелем;
- корпоративными уведомлениями.

### 10. AI-ассистент

В кейсе описан AI-модуль и чат-ассистент, но в текущем коде отдельный AI-ассистент пока не реализован.

### 11. История изменений

Пока нет хранения истории изменений рабочего графика.

### 12. Автоматические тесты

Пока нет полноценного исполняемого набора автотестов. Есть сценарии, которые можно перенести в backend tests и frontend tests.

## Что нужно уточнить у команды

1. Нужно ли делать единый backend endpoint `/api/worktime/overview` для frontend?
2. Frontend должен работать только с backend или mock-режим оставить для демо?
3. Какие данные считаем основными: backend JSON или frontend mockData?
4. Какие роли реально показываем на защите?
5. Будет ли демонстрация Swagger backend?
6. Нужно ли делать загрузку CSV/JSON в рамках хакатона?
7. Нужно ли добавлять автоматические тесты до защиты?
8. Какие screenshots/interface screens попадут в презентацию?
9. Будет ли отдельный AI-ассистент или только explainable recommendations?
